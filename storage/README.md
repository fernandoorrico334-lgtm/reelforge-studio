# Storage

- `assets/`: arquivos enviados pelo usuario, organizados por projeto.
- `references/`: reels locais de referencia usados apenas para extrair padroes
  editoriais, nunca para versionar ou reaproveitar conteudo.
- `renders/`: saidas exportadas localmente pelo worker quando a fase de render chegar.
- Ambas as pastas ficam versionadas apenas com placeholders; o conteudo operacional real deve ficar fora do Git.
